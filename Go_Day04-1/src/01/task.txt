TARGET: реализовать аутентификацию по сертификату для сервера, а также тестового клиента, который сможет запрашивать ваш API, 
используя самоподписанный сертификат и локальный центр безопасности для его «проверки» с обеих сторон.

TASKs
1)  вам понадобится локальный «центр сертификации» для управления сертификатами.
    Для нашей задачи minica кажется достаточно хорошим решением. 

2)  Итак, поскольку речь идет о полноценной взаимной TLS-аутентификации, вам придется сгенерировать две пары 
    сертификатов/ключей - одну для сервера и одну для клиента. Minica также сгенерирует для вас файл CA под названием minica.pem, 
    который вам нужно будет каким-то образом подключить к вашему клиенту (ваш автоматически сгенерированный сервер уже должен 
    поддерживать указание файла CA, а также key.pem и cert.pem через параметры командной строки). 
    Кроме того, при генерации сертификата может потребоваться использовать домен вместо IP-адреса, поэтому в примерах ниже мы 
    будем использовать «candy.tld».

3)  Имейте в виду, что, поскольку вы используете собственный локальный CA, вы, скорее всего, не сможете запрашивать API с 
    помощью cURL, веб-браузера или такого инструмента, как Postman, без тюнинга.

4)  Ваш тестовый клиент должен поддерживать флаги '-k' (принимает двухбуквенную аббревиатуру типа конфет), '-c'
    (количество конфет для покупки) и '-m' (количество денег, которые вы «отдали автомату»). Таким образом, «запрос на покупку» 
    должен выглядеть следующим образом:

~$ ./candy-client -k AA -c 2 -m 50
