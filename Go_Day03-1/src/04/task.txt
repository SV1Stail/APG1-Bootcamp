
1)  реализовать конечную точку API http://127.0.0.1:8888/api/get_token, 
    единственной целью которой будет генерировать токен и возвращать его

2)  Don't forget to set header 'Content-Type: application/json'.
3)  защитить конечную точку /api/recommend с помощью промежуточного
     ПО JWT, которое будет проверять валидность этого токена.
4)  при запросе этого API из браузера он теперь должен выдать 
    ошибку HTTP 401, но работать, если в заголовке Authorization: Bearer <token>
    заголовок указан клиентом (вы можете проверить это с помощью cURL или Postman).
